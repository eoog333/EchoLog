EchoLog

대화하듯이 음성으로 편하게 녹음하는 하루 기록용 앱입니다.

프로젝트 개요

EchoLog는 사용자의 음성을 텍스트로 전사하고, 발화 시각(Timestamp)을 기준으로 기록을 정렬하는 서비스입니다. RTZR STT API의 전처리 기능과 자체 후처리 파이프라인을 결합하여 가독성 있는 형태로 텍스트를 구성합니다.

* 본 애플리케이션은 기능 검증용 데모 버전으로, 데이터베이스 저장 및 기록 보존 기능은 제공하지 않습니다. (새로고침 시 데이터가 초기화됩니다.)

사용 흐름

1. 프론트엔드에서 음성 녹음 시작 및 완료
2. 음성 데이터를 WAV 포맷으로 백엔드 전송
3. 백엔드에서 비동기로 전사 처리 및 타임라인 결과 반환

기술 분석

아키텍처 구성

  • Frontend (React/Vite): Web Audio API를 사용하여 브라우저에서 직접 오디오를 수집 및 WAV 포맷 인코딩 처리.
  • Backend (FastAPI): 비동기 폴링(Polling) 방식을 통해 STT 처리를 요청 및 대기.

후처리 파이프라인 알고리즘

  • 간투어 필터 적용: 전사 결과에서 불필요한 추임새 제거.
  • 중복 발화 제거 (Dedup): 직전 발화와의 시간 간격이 3초 이내이고, 텍스트 유사도가 0.8 이상인 경우 중복으로 판단하여 제거.
  • 타임라인 정렬 및 문단화: 전사된 문장의 실제 start_at 속성을 기준으로 정렬하며, 시간 표현 문맥을 바탕으로 단락 구분.

사전 준비

  • Python: 3.11+ 설치 필요
  • Node.js: 20.19+ 또는 22.12+ 설치 필요
  • RTZR API 키: 리턴제로 개발자 사이트 가입 및 API 발급 필요

실행 단계

운영체제에 맞는 탭을 선택하고 명령어를 실행해 주세요. 가상환경을 활성화하려면 루트 폴더가 아닌 backend 디렉토리 내부로 반드시 이동해야 합니다. 

git clone https://github.com/eoog333/EchoLog-Demo.git
cd EchoLog-Demo

# 1. 백엔드 실행 (터미널 1)
cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# .env 파일에 RTZR API 키 입력
uvicorn app.main:app --reload

# 2. 프론트엔드 실행 (새로운 터미널 2)
cd ../frontend
npm install
npm run dev

환경변수 설정 (`.env`)

backend/.env 파일에 다음 값들을 설정합니다. 

RTZR_CLIENT_ID=your_client_id
RTZR_CLIENT_SECRET=your_client_secret
LLM_API_KEY=          # 향후 LLM 연동 확장용 — 현재 미사용
