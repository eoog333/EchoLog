EchoLog는 음성 회고를 텍스트로 단순 변환하는 것을 넘어, 인공지능 STT 전처리 기능과 정교한 시간순 정렬 알고리즘을 사용해 가독성 높은 하루 타임라인을 구성합니다.

구동방식 체험하기 로컬에 배포하기

프로젝트 해결 과제

기존의 음성 메모 방식과 텍스트 전사(STT)가 가지는 한계를 극복하고 개발자와 사용자에게 최고의 재현성과 가독성을 선물합니다.

🎤

자연스러운 음성 필터링

RTZR STT의 간투어 필터(Disfluency Filter)를 연동하여 말할 때 무의식적으로 뱉는 "어, 음, 그" 같은 추임새를 제거하고 알맹이 발화만 정제합니다.

🕒

시간 흐름 보존

시간 정보가 포함된 말소리를 감지하여, 발화의 실제 시간 순서대로 문장을 자동 그룹화하고 타임라인 형태의 회고 기록을 구성합니다.

🔗

100% 실행 재현성

맥과 윈도우 환경에 최적화된 독립 가상환경 매뉴얼을 준수하여, 저장소 클론 후 개발자가 즉시 에러 없이 로컬 실행을 보장합니다.

가상 시뮬레이터 (UX)

로컬 실행 및 배포 (재현성)

운영체제 맞춤형 명령어를 지원하여, 개발자의 가상환경 구동 오류를 방지하고 재현을 보장합니다.

⚠️ 중요: 백엔드 구동 시 루트폴더가 아닌 backend 디렉토리 내부로 들어간 뒤 가상환경을 실행해야 정상 작동합니다.

source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# .env에 발급받은 RTZR API 키 입력 후 구동

기술 분석 및 파이프라인

백엔드 후처리 엔진 및 RTZR STT API가 결합한 오디오 데이터 처리 워크플로우를 분석합니다.

시스템 전체 플로우

1

Web Audio API (Frontend)

브라우저 환경에 구애받지 않도록 미디어 버퍼를 수집하고 WAV(audio/wav)로 포맷 변환 후 전송

2

FastAPI (Backend)

멀티파트 파일을 받아 RTZR API로 전사 요청 및 완료될 때까지 비동기 3초 간격 롱 폴링(Polling) 수행

3

후처리 엔진 (Post-Processor)

발화 간격 및 텍스트 유사도를 비교하여 0.8 이상의 중복을 걸러내고 타임라인별 그룹 재배열

후처리 파이프라인 핵심 알고리즘

1. 중복 제거 (Dedup)

직전 발화와의 시각 차가 3초 이내이고 의미 유사도가 0.8 이상일 때, 중복된 문장을 자동 탈락시켜 간결함을 유지합니다.

2. 시간대별 문단화

문장 앞머리에 위치하는 시간 표현(예: '아침', '오후 3시')을 파싱해 시간대별 하루 흐름을 보존하며 문단을 가릅니다.

3. 자연스러운 문체 정리

의미가 끊기지 않도록 문장 끝 단위로 결합하여, 문단마다 2~3개의 정돈된 단락 흐름을 제공합니다.
