# 1. 프로젝트 소개

## 프로젝트 개요

EchoLog는 사용자가 친구에게 이야기하듯 하루를 자유롭게 말하면
**RTZR STT API**를 이용해 음성을 전사하고, 전사 결과를 읽기 쉬운
**회고(Reflection)** 형태로 재구성하는 웹 애플리케이션입니다.

이 프로젝트의 목적은 AI 일기 서비스를 만드는 것이 아니라 **RTZR STT
API가 생성한 전사 결과를 활용하여 사람이 다시 읽기 좋은 기록으로
재구성하는 후처리 파이프라인을 구현하는 것**입니다.

**핵심 가치**

> 말하기 기록을 읽기 쉬운 기록으로 바꾼다.

## 네이밍 배경 (Why EchoLog?)

**Echo (메아리, 음성) + Log (기록)**
내가 무심코 뱉은 말(Echo)이 정제되어 의미 있는 명확한 기록(Log)으로 되돌아온다는 뜻을 담고 있습니다. 음성을 텍스트로 기록하는 STT 기술의 본질과 서비스의 정체성을 직관적으로 표현합니다.

------------------------------------------------------------------------

# 2. 해결하려는 문제

- 하루를 기록하고 싶지만 타이핑은 번거롭다.
- 음성 메모는 자연스럽지만 다시 듣기 어렵다.
- STT 결과는 구어체라 그대로 읽기에는 불편하다.

EchoLog는 이러한 문제를 해결하기 위해 STT 결과를 후처리하여
자연스러운 회고 형태로 제공합니다.

# 5. RTZR API 활용 근거

  기능                 사용 목적
  -------------------- -----------------------
  Authentication       API 인증
  Batch STT            음성 전사
  Polling              비동기 전사 완료 확인
  Paragraph Splitter   문단 단위 후처리
  Disfluency Filter    추임새 제거
  start_at             사건 순서 유지

**설계 원칙**

- RTZR STT가 프로젝트의 중심이다.
- LLM은 문체를 다듬는 보조 역할만 수행한다.

# 6. Transcript Post Processing

## 처리 흐름

``` text
Transcript

↓

Paragraph Splitter

↓

문단 리스트 생성

↓

시간 순서 유지

↓

중복 내용 제거

↓

사건 단위 정리

↓

LLM (문체만 다듬기)

↓

Reflection
```

## 단계별 역할

### 1. Paragraph Splitter

- 긴 구어체를 의미 단위 문단으로 분리

### 2. 시간 순서 유지

- 전사 순서를 그대로 유지하여 하루의 흐름 보존

### 3. 중복 제거

- 반복되는 내용을 정리

### 4. 사건 단위 정리

- 하루 동안 발생한 사건을 읽기 쉬운 형태로 재배열

### 5. LLM

- 표현을 자연스럽게 다듬되 새로운 사실은 생성하지 않음
